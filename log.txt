## Expert failure analysis summary

The expert comparison found five repeated failure patterns:

```text
1. Sentence-before under-extraction is the main issue.
2. Role classifier over-fires generic "possible X definition" outputs.
3. Symbol-assisted fallback emits symbol-bags instead of whole-equation meanings.
4. Subequation group-intro is not propagated.
5. Equation-internal \text{...} labels are not read.
```

Important diagnostic:

```text
Sentence-before carries about 78% of real equation meanings in the reference audit.
```

Therefore, the architecture is correct, but the extraction priority is wrong.

The current pipeline should prioritize real local prose before falling back to generic role classifier or symbol-assisted fallback.

## Task 1 — Strengthen sentence-before extractor

Make the sentence immediately before the equation the primary high-confidence path.

Add or improve regex/rule patterns for cue phrases such as:

```text
is defined as
are defined as
is given by
are given by
is described by
are described by
is governed by
is expressed as
can be written as
may be written as
takes the form
satisfies the ... equation
is implemented using
is represented as
we represent ... as
we define ... as
we introduce ... as
the equation of motion
the master equation
the Hamiltonian
the energy
the action
the probability
the self-attention score
the imbalance
```

Extract a short whole-equation meaning.

Good examples:

```text
energy of the string
action of the relativistic string
master equation for the quantum channel
Hamiltonian decomposition into local terms
linear cross-entropy benchmark score
imbalance between even and odd sites
n-th evolution-step operator
time-averaged probability
```

Bad examples:

```text
given by
defined as
as follows
this equation
relation among symbols
```

The extractor should require adjacency: the sentence should directly introduce the displayed equation or be the nearest previous sentence.

## Task 2 — Strengthen sentence-after extractor

Support cases where the equation role is explained immediately after the equation.

Useful cue phrases:

```text
describes
represents
corresponds to
is referred to as
which is called
which gives
this expression gives
this equation describes
this relation shows
```

Only use these when they describe the whole equation.

Do not use after-equation text that only defines one symbol unless it can support a whole-equation role.

## Task 3 — Add strict equation-internal text-label extraction

Extract short labels from equation LaTeX commands such as:

```text
\text{...}
\textrm{...}
\mathrm{...}
\mbox{...}
```

Use the label only if it clearly names the whole-equation role.

Accept examples:

```text
absorbing boundary condition
boundary condition
transition probability
```

Reject/downgrade examples:

```text
in N
in S
case 1
for x > 0
with
vs
v.s.
string
where
```

Do not accept units, region labels, case labels, or plain connective text as equation meanings.

## Task 4 — Add equation-group carryover

Support multi-equation and subequation groups.

Examples:

```text
The solutions are defined as:
Eq. 2a
Eq. 2b
```

```text
A straightforward calculation yields:
Eq. 5
Eq. 6
Eq. 7
```

General rule:

If a sentence introduces a group of consecutive displayed equations, allow nearby subequations or consecutive equations to inherit a review-level meaning from that group intro.

Preserve subequation labels such as:

```text
1a
1b
2a
2b
3a
3b
3c
```

Important:

Group-intro meanings should normally be `needs_manual_review`, unless each individual subequation has explicit local prose.

Do not over-propagate across unrelated equations. Restrict carryover to same display block or very close consecutive equations.

## Task 5 — Demote role classifier

Role classifier should be last-resort only.

It must not overwrite:

```text
equation_internal_text_label
sentence_before
sentence_after
equation_group_intro
```

Generic role outputs should usually be low-confidence and review-only:

```text
possible Hamiltonian definition
possible Liouvillian definition
possible state evolution equation
possible probability definition
possible normalization condition
possible operator definition
```

Do not allow broad role labels to become accepted unless supported by explicit local prose.

Especially tighten normalization detection.

Only produce normalization-related meaning if local context or equation text includes:

```text
normalization
normalized
normalization factor
sum to one
probability sums
```

Do not classify arbitrary sums, fractions, or prefactors as normalization.

## Task 6 — Demote/block symbol-assisted fallback

Symbol-assisted fallback must never produce symbol-bag meanings.

Block meanings like:

```text
relation among rho, Lambda, and t
relation among symbols
relation among analog variable, speed limit, and length
rho means density matrix
H means Hamiltonian
```

These are not whole-equation meanings.

If symbol-assisted fallback is used at all:

```text
confidence: low
quality_bucket: needs_manual_review
source_method: symbol_meaning_assisted
```

Use only accepted symbol meanings, not review-needed or unresolved symbol meanings.

Safer optional fallback:

If equation is clearly `LHS = RHS` and LHS has an accepted local symbol meaning, output:

```text
expression for <LHS meaning>
definition of <LHS meaning>
```

But keep it `needs_manual_review` unless explicit local prose supports it.

## Task 7 — Quality gate improvements

## Task 8 — Prefer local prose over section heading

Section heading alone is not enough for accepted meaning.

Local prose wins over section heading.
